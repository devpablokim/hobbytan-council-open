# 🔄 HOBBYTAN-COUNCIL 5-Cycle Verification Protocol (The 20-Step Report System)

**제정일**: 2026-02-07
**상태**: **MANDATORY (강제 사항)**

---

## 1. 개요
모든 주요 개발/수정 작업은 **5번의 사이클(Iteration)**을 거치며, 각 사이클은 **4단계 보고**로 구성된다.
따라서 하나의 태스크는 총 **20단계**의 보고 절차를 통해 완벽하게 검증된다.

## 2. 5-Cycle 구조
- **Cycle 1**: Core Functionality (기능 구현 및 배포)
- **Cycle 2**: Visual Upgrade (디자인 및 에셋 강화)
- **Cycle 3**: Persuasion & Logic (카피라이팅 및 설득 구조 강화)
- **Cycle 4**: UX Polish (모바일 최적화, 로딩 속도, 디테일)
- **Cycle 5**: Final Integrity Check (최종 검수 및 그랜드 오픈)

## 3. 각 Cycle별 4단계 보고 절차 (Mandatory Steps)

각 단계 완료 시, 반드시 **Telegram**으로 보고하며, **관련 링크(회의록, 코드, 배포 URL)**를 첨부해야 한다.

### Step 1. 기획 회의 (Planning)
- **행동**: 의회를 소집하여 이번 사이클의 목표와 상세 스펙을 확정한다.
- **보고**: "Cycle X 기획 회의 완료. [회의록 링크] [기획서 링크]"

### Step 2. 개발 착수 (Development Start)
- **행동**: 확정된 기획안을 바탕으로 코드 작성을 시작한다. (보안 검사 필)
- **보고**: "Cycle X 개발 착수. [수정 대상 파일 목록]"

### Step 3. 배포 완료 (Deployment)
- **행동**: 코드를 배포하고 라이브 환경에서 확인한다.
- **보고**: "Cycle X 배포 완료. [배포 URL] [커밋 링크]"

### Step 4. 자체 QA 및 회고 (QA & Retro)
- **행동**: 배포된 결과물을 13인의 TAN이 뜯어보고 문제점을 찾는다.
- **보고**: "Cycle X QA 결과 보고. [발견된 버그/개선점 리스트] [QA 회의록 링크]"

---

## 4. 실행 규칙
1.  이 프로토콜은 유저가 "그만하라"고 할 때까지 멈추지 않는다.
2.  단계를 건너뛰거나(Skip), 요약 보고(Summary)를 하는 것은 금지된다.
3.  모든 보고에는 **'증거(Artifact)'**가 동반되어야 한다.

**"검증되지 않은 결과물은 쓰레기다."**
