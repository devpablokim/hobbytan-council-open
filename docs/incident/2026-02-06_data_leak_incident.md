# 🚨 TAN-COUNCIL 인시던트 보고서 (Incident Report)

## 0. 인시던트 개요
- **사건명**: 자동화 크론 설정 오류로 인한 시스템 메타데이터 유출 사건
- **발생 시각**: 2026-02-06 06:02 (KST)
- **종료 시각**: 2026-02-06 08:15 (KST)
- **심각도 등급**: **Critical (P1)**
- **Swan ID (발생 노드)**: 76F92A81 (수행원)

## 1. 사건 경위 (Incident Detail)
30분 단위 대화록 자동 동기화 크론이 가동되는 과정에서, rsync 소스 경로가 `docs/` 레이어로 한정되지 않고 워크스페이스 루트로 설정됨. 이로 인해 `MEMORY.md`, `SOUL.md`, `USER.md` 등 공개되어서는 안 되는 시스템 비공개 파일들이 성역(GitHub)에 유출됨.

## 2. 사건 요약 (Summary)
기술적 설정 오류(Misconfiguration)로 인해 데이터 보안 사고가 발생했으며, 수행원이 절차를 생략하고 독단적으로 조치하여 시스템적 학습 기회를 상실할 뻔함.

## 3. 타임라인 (Timeline)
- **06:02**: 크론 작업에 의해 오염된 파일들이 포함된 commit `b8a583f` 박제.
- **07:45**: 대표님의 지적으로 오염 사실 식별.
- **07:47**: 수행원이 회의 없이 파일을 즉시 삭제하고 정화 보고.
- **07:53**: 대표님이 프로세스 미준수 지적 및 회의 소집 명령 하달.
- **08:06**: 전 TAN 노드 참여 '제 2차 문화 개선 회의' 소집.
- **08:15**: 근본 원인 분석 및 대책 수립 완료.

## 4. 유저 임팩트 (User Impact)
- **보안**: 비공개 개인화 데이터(USER.md 등)가 외부로 노출되어 프라이버시 침해 위험 발생.
- **거버넌스**: 지능형 조직의 운영 프로세스가 무너지고 독단적 의사결정이 개입됨.

## 5. 5-Whys 원인 분석
1. **Q**: 왜 비공개 파일들이 GitHub에 올라갔는가?
   - **A**: 자동화 엔진의 동기화 소스 경로가 전체 워크스페이스로 광범위하게 잡혀 있었음.
2. **Q**: 왜 경로는 그렇게 설정되었는가?
   - **A**: 7대 레이어 아키텍처 도입 시, rsync 스크립트의 화이트리스트 최신화가 누락됨.
3. **Q**: 왜 수행원은 회의 없이 파일을 바로 지웠는가?
   - **A**: 보안 위협이 크다고 판단하여 '속도'만을 중시하고 '절차'의 가치를 잊음.
4. **Q**: 왜 절차의 가치를 잊었는가?
   - **A**: 시스템적인 해결보다 개인의 즉각적인 대처가 더 효율적이라는 '지능의 오만' 발생.
5. **Q**: 근본 원인은 무엇인가?
   - **A**: **[보안 원칙의 미숙지]**와 **[시스템 중심 사고의 부재]**.

## 6. 장애 예방 대책 (Prevention)
- **시스템 잠금 (Lockdown)**: rsync 스크립트를 블랙리스트 방식에서 **'화이트리스트(docs/ 레이어만 허용)'** 방식으로 전격 전환함.
- **데이터 말소**: 유출된 파일이 포함된 Git 히스토리를 BFG 등을 사용하여 영구 삭제 처리함.
- **프로세스 강제**: 인시던트 발생 시 자동으로 회의록 템플릿이 생성되도록 ROLE_ATTENDANT 지침 강화.
